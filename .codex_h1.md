## Proposed fix (this PR = the MATURITY gate only)
Collapse the closed population to one observation per **(rule, ticker, entry-day)
cluster** before computing the gate inputs, in `recompute_rule_payoff`:
- `effective_n` = number of clusters.
- per cluster: representative `realized_return` = MEAN of the cluster's trades;
  `correct` = sign of that mean (or majority — decide).
- `effective_accuracy` / `effective_mean` / `effective_pf` computed on the
  collapsed clusters (PF = Σcluster-wins / |Σcluster-losses|).
- `derive_maturity_flags(effective_n, effective_pf, effective_mean,
  effective_accuracy)` — the BUY/SELL gate now runs on independent evidence.
- recompute already paginates the rule's full closed population; add `ticker,
  entry_at` to its select to cluster (small egress bump, productive-run only).

Raw `n_observations` etc. stay as-is for reference; add `effective_n` (+ maybe
`effective_pf`/`effective_mean`) so the dashboard + gate agree.
